Что такое Яхонт

Яхонт — это Open Source библиотека нового поколения высокоуровневых решений для разработчиков Android.

Ключевые характеристики и особенности:

  • Загрузка и отображение на экране данных из Интернета + определение местоположения: всего 1-2 строки кода (вместо обычных 300-500 и более строк)
  • Совершенно новые возможности: добавление обработчиков событий к любым объектам БЕЗ изменения их исходного кода (именно благодаря этому стало возможным упомянутое выше)
  • Набор универсальных и готовых к использованию компонентов: не требующий настройки кеш, автоматический запрос необходимых разрешений (runtime permissions) и многое другое
  • Поддержка новейших технологий (Dagger 2, Rx Java 2, новый Google Location API)

Цель проекта — максимально упростить и удешевить разработку Android-приложений, одновременно радикально (в разы) повысив качество.

Перспективы развития — интеграция с системами искусственного интеллекта (для создания полноценных Android-приложений с минимальным участием человека).

Yakhont: high-level Android components for data loading, location, lifecycle callbacks and many more

Yakhont is an Android high-level library offering developer-defined callbacks, loader wrappers and adapters, fully automatic cache, location-awareness, dynamic permissions handling, lifecycle debug classes, advanced logging and many more helpful developer-oriented features.

In short, the features the Yakhont provides are:

  • developer-defined callbacks to customize lifecycle of Activities and Fragments - and even without changing their sources
  • powerful (but very easy in use) data loaders and adapters
  • automatic (but fully customizable) data binding
  • self-configurable transparent cache which adjusts database structure 'on the fly'
  • out-of-the-box location awareness: just annotate your Activity and you're done
  • dynamic permissions handling, powered by user-defined callbacks
  • debug classes with strict mode and lifecycle logging - for all kinds of Activities and Fragments (can be enabled even for 3rd-party components via simple, but effective Yakhont preprocessor)
  • advanced logging with e-mail support (auto-disabled in release builds) and more.

The goal of the project is to simplify and make the development of Android applications as cheap as possible, at the same time greatly improving the quality.

The development prospect is an integration with artificial intelligence systems (for creating high-grade Android applications with minimal human participation).

Что предоставляет Яхонт:

Модуль Callback

  • возможность расширения обработчиков событий жизненного цикла (lifecycle callbacks) для Activities и Fragments (но не только)

Модуль местоположения (Location)

  • очень прост в использовании - достаточно пометить Activity с помощью Annotation
  • поддержка как нового, так и старого Google Location API
  • поддержка _Rx_
  • автоматический запрос разрешений (dynamic permissions)
  • функции преобразования в DMS формат (пример результата: _5°31′08″ N 87°04′18″ W_)

Модуль загрузки данных (Loader)

  • встроенный кеш
  • встроенный swipe-to-refresh
  • полностью асинхронное выполнение
  • время выполнения никогда не превысит заданного предельного значения (timeout)
  • поддержка _Retrofit_
  • поддержка _Rx_
  • поддержка смены ориентации устройства
  • полностью настраиваемый графический интерфейс (GUI)
  • возможность прервать загрузку (с выводом диалога подтверждения - также полностью настраиваемого)
  • возможность расширения (добавления поддержки не-Retrofit компонентов)

Модуль Retrofit

  • поддержка Retrofit и Retrofit 2
  • поддержка HTTP заголовков (headers) и cookies (только для Retrofit 2)
  • локальные клиенты (доступны в демо-приложениях)

Модуль Rx

  • поддержка RxJava и RxJava 2
  • встроенные обработчики ошибок (error handlers)
  • встроенная поддержка _SafeSubscriber_

Модуль адаптера (Adapter)

  • полностью автоматическое (но с возможностью тонкой настройки) связывание данных (data binding)
  • поддержка встроенного кеша
  • поддержка _RecyclerView_, _ListView_ и _GridView_

Модуль кеша (Cache)

  • полностью автоматический и прозрачный в использовании
  • создание таблиц и добавление колонок "на лету"
  • поддержка BLOB
  • поддержка транзакций (включая групповое добавление - bulk insert)
  • поддержка копирования базы данных (для отладки)

Модуль разрешений (Permissions)

  • полностью автоматический запрос разрешений и обработка ответов от системы: достаточно определить "on granted" и / или "on denied" обработчики (callbacks)
  • полностью настраиваемый графический интерфейс (GUI)

Модуль графического интерфейса (GUI)

  • progress и alert диалоги (полностью настраиваемые через Dagger 2)
  • независимые от контекста (_Context_) _Snackbar_'ы и _Toast_'ы (необходимый контекст определяется автоматически)

Модуль Dagger 2

  • настройка графического интерфейса (GUI) и клиента определения местоположения (location)

Модуль логирования (Logging)

  • автоматическое отключение в release-сборках
  • добавление stack traces для любых записей (настраивается)
  • возможность отправки лог-файлов и снимков экрана по электронной почте (достаточно встряхнуть устройство); поддерживаются любые команды _logcat_
  • возможность расширения (подключение компонентов логирования от сторонних разработчиков)

Модуль System

  • возможность определения ухода приложение в фоновый режим (background)
  • возможность определения текущей Application, Activity и View по умолчанию (для данной Activity)
  • поддержка работы с электронной почтой
  • поддержка работы с архивами (zip)

Модуль Reflection

  • улучшенный поиск методов: поддержка auto unboxing, supertype и null
  • выполнение с поддержкой (либо без) исключительных ситуаций (exceptions)
  • удобные методы для работы с полями (fields) - получение и присваивание значений

Модуль отладки (Debug)

  • поддержка логирования жизненного цикла Application, Activities и Fragments (с возможностью добавления компонентов от сторонних разработчиков)
  • более 40 готовых классов (только в версии _full_)
  • поддержка StrictMode

Модуль Preprocessor

  • создание исходного кода для _support_ и _full_ версий (включая классы для отладки, с возможностью настройки)

Модуль Weaver

  • возможность изменения байт-кода откомпилированных классов приложения, полностью настраивается

Yakhont: the list of available features

Callback components

  • developer-defined callbacks to customize lifecycles of Activities and Fragments

Location component

  • extremely easy to use - just annotate your Activity
  • both new and old Google Location API support
  • _Rx_ support
  • auto handling of dynamic permissions
  • build-in conversion to DMS format (something like _5°31′08″ N 87°04′18″ W_)

Loader component

  • build-in cache
  • build-in swipe-to-refresh
  • fully asynchronous
  • forced timeouts
  • _Retrofit_ support
  • _Rx_ support
  • device orientation changing support
  • fully customizable progress GUI
  • possibility to cancel data loading (also with customizable GUI)
  • extendable (support for non-Retrofit components can be added)

Retrofit components

  • both Retrofit and Retrofit 2 support
  • HTTP headers and cookies (Retrofit 2 only) support
  • local clients (in demo)

Rx components

  • both RxJava and RxJava 2 support
  • build-in error handlers
  • _SafeSubscriber_ wrappers

Adapter component

  • automatic (but fully customizable) data binding
  • build-in cache support
  • _RecyclerView_, _ListView_ and _GridView_ support

Cache component

  • automatic and absolutely transparent
  • creates tables and adds columns "on the fly"
  • BLOB support
  • transactions support (including bulk insert)
  • database copy support (for debugging)

Permissions component

  • completely auto handling of permission requests and system responses: just define "on granted" and / or "on denied" callbacks
  • fully customizable GUI

GUI component

  • progress and alert dialogs (fully customizable via Dagger 2)
  • context-independent _Snackbars_ and _Toasts_ (required _Context_ defines automatically)

Dagger 2 component

  • GUI and location clients customization

Logging component

  • auto-disabled in release builds
  • stack traces for logging records (customizable)
  • supports sending logs and screenshots via e-mail (just shake device); any _logcat_ command can be used
  • extendable (supports 3rd-party logging components)

System component

  • possibility to check either application coming to the foreground or going to the background
  • methods to find out current Application, Activity and default View for the given Activity
  • e-mail support
  • zip support

Reflection component

  • advanced method finding: auto unboxing, supertypes and nulls support
  • invocation with or without exceptions
  • useful setters and getters for fields

Debug components

  • Application, Activities and Fragments lifecycle logging support (can be enabled even for 3rd-party components)
  • more than 40 build-in debug classes (in _full_ version only)
  • StrictMode support

Preprocessor component

  • generates sources for _support_ and _full_ versions (including debug classes, customizable)

Weaver component

  • manipulates the compiled Java bytecode, fully customizable
